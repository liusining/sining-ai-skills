# Agent Create

Create new OpenClaw agents via the `openclaw agents add` CLI with proper workspace and git hygiene.

## Skill directory layout

```
agent-create/
  SKILL.md
  reference/
    openclaw/          ← git submodule of github.com/openclaw/openclaw
      docs/
        reference/templates/         ← English workspace templates
        zh-CN/reference/templates/   ← Chinese workspace templates
```

The `reference/openclaw` submodule contains official workspace template files (`AGENTS.md`, `SOUL.md`, `USER.md`, `BOOTSTRAP.md`, `HEARTBEAT.md`, `TOOLS.md`, etc.) in both English and Chinese.

**Resolve the skill directory** from the location of this SKILL.md file. All relative paths below (e.g. `reference/openclaw/...`) are relative to the skill directory.

## Required inputs

Ask the user before creating:

| Input | Question | Required | Default | Example |
|-------|----------|----------|---------|---------|
| Agent ID | Short identifier (lowercase, no spaces) | Yes | — | `research` |
| Agent name | Display name | No | same as ID | `Ada` |
| Model | Model ID or alias | No | gateway default | `anthropic/claude-sonnet-4-6` |
| Language | Chinese (zh) or English (en) workspace files? | Yes | en | `zh` |
| Channel binding | Which channel(s) to bind | No | none | `telegram:research` |

If unclear, stop and ask.

## Pre-flight checks

1. Verify `openclaw` CLI is available:
   ```bash
   openclaw --version
   ```
2. List existing agents to avoid ID collisions:
   ```bash
   openclaw agents list
   ```
3. If the requested agent ID already exists, warn the user and stop.
4. Verify the reference submodule exists:
   ```bash
   ls <skill-dir>/reference/openclaw/docs/reference/templates/AGENTS.md
   ```
   If missing, initialize it:
   ```bash
   cd <skill-dir> && git submodule update --init --recursive
   ```

## Create the agent

Use `openclaw agents add` with `--non-interactive` to avoid TTY prompts:

```bash
openclaw agents add <agent-id> \
  --workspace ~/.openclaw/workspace-<agent-id> \
  --model <model-id> \
  --non-interactive \
  --json
```

If channel bindings were requested, add `--bind` flags:

```bash
openclaw agents add <agent-id> \
  --workspace ~/.openclaw/workspace-<agent-id> \
  --model <model-id> \
  --bind telegram:<accountId> \
  --non-interactive \
  --json
```

### Handle interactive fallback

If `--non-interactive` fails or is not supported by the installed version, use PTY mode (`pty: true`) and answer the interactive prompts programmatically.

## Post-creation: language localization

If the user chose **Chinese (zh)**:

1. Set the template source directory:
   ```
   TEMPLATES=<skill-dir>/reference/openclaw/docs/zh-CN/reference/templates
   ```

2. Copy Chinese template files over the CLI-generated English ones.
   Replace **all** of these files in `~/.openclaw/workspace-<agent-id>/`:
   - `AGENTS.md`
   - `SOUL.md`
   - `USER.md`
   - `BOOTSTRAP.md`
   - `HEARTBEAT.md`
   - `TOOLS.md`

   ```bash
   WORKSPACE=~/.openclaw/workspace-<agent-id>
   for f in AGENTS.md SOUL.md USER.md BOOTSTRAP.md HEARTBEAT.md TOOLS.md; do
     cp "$TEMPLATES/$f" "$WORKSPACE/$f"
   done
   ```

3. **Do NOT replace** `IDENTITY.md` — keep the English version generated by the CLI.

If the user chose **English (en)**: no replacements needed — the CLI-generated files are already in English.

## Post-creation: git hygiene

OpenClaw's `agents add` command runs `git init` inside the new workspace automatically. This is **not** desired — git should be managed at the `~/.openclaw` level, not per-workspace.

After creation, always:

1. **Remove the per-workspace `.git` directory** if it exists:
   ```bash
   rm -rf ~/.openclaw/workspace-<agent-id>/.git
   ```

2. **Ensure `~/.openclaw` itself has a git repo** (the parent for all workspaces):
   ```bash
   if [ ! -d ~/.openclaw/.git ]; then
     cd ~/.openclaw && git init
   fi
   ```

3. **Verify cleanup**:
   ```bash
   ls -d ~/.openclaw/workspace-<agent-id>/.git 2>/dev/null && echo "ERROR: .git still exists" || echo "OK: no per-workspace .git"
   ```

## Post-creation: verify

1. Confirm the agent appears in the list:
   ```bash
   openclaw agents list
   ```

2. Show the workspace contents:
   ```bash
   ls -la ~/.openclaw/workspace-<agent-id>/
   ```

3. If channel bindings were set, verify them:
   ```bash
   openclaw agents bindings
   ```

## Post-creation: optional setup

If the user wants to customize the agent's personality or workspace files, offer to:

- Edit `SOUL.md` — agent persona and voice
- Edit `USER.md` — info about the human
- Edit `IDENTITY.md` — agent name, emoji, avatar
- Edit `AGENTS.md` — workspace conventions

These files are in `~/.openclaw/workspace-<agent-id>/`.

## Error handling

| Error | Cause | Fix |
|-------|-------|-----|
| Agent ID already exists | Duplicate ID | Choose a different ID or delete the existing agent first |
| `--non-interactive` requires `--workspace` | Missing workspace flag | Always pass `--workspace` explicitly |
| Gateway not running | Agent registration needs gateway | Run `openclaw gateway start` first |
| Channel binding fails | Invalid channel or account ID | Check `openclaw channels list` for valid channels |
| Reference submodule missing | Submodule not initialized | Run `git submodule update --init` in the skill directory |
| Chinese template file missing | Submodule out of date | Run `git submodule update --remote` in the skill directory |

## What NOT to do

- **Never** create workspaces and config entries manually via `exec` / file writes — always use `openclaw agents add`
- **Never** leave per-workspace `.git` directories — remove them after creation
- **Never** skip the post-creation verification step
- **Never** replace `IDENTITY.md` with the Chinese version — keep it in English
